Processo

Objetivo: Aplicar os três métodos de inserção de CSS, garantindo ao mesmo tempo uma organização profissional de pastas e arquivos no seu repositório.

Tarefa 1 - Configuração do Ambiente e Estrutura de Pastas

O objetivo aqui é criar a base organizada para evitar links quebrados e facilitar a manutenção

  1. Crie um repositório no GitHub: Nomeie-o como portifolio-info1-2026, certifique de deixar "on" a opção "README" e que ele seja Público.
  2. Organize as pastas no seu computador:
    • Dentro da pasta do projeto, crie a seguinte estrutura (use sempre letras minúsculas e hífens no lugar de espaços).
    • Crie o arquivo index.html na raiz (pasta principal) do seu projeto
    • Criar uma pasta chamada "assets": Dentro dela, crie duas subpastas chamadas: "css" e "images".
    • Criar uma pasta chamada "html": Dentro dela, crie três subpastas chamadas: "css-inline", "css-interno" e "css-externo" para organizar das tarefas.

Tarefa 2: Praticando CSS Inline

O objetivo aqui é criar um arquivo com CSS inline é aplicado diretamente na tag HTML para mudanças pontuais.

  1. Crie o arquivo css_inline.html dentro da pasta html/css-inline/.
  2. No Codespace do github, use o atalho html:5 para gerar a estrutura básica.
  3. Crie o arquivo index.html na raiz (pasta principal) do seu projeto
  4. No corpo do documento (elemento body), adicione um título h1.
  5. Aplique o estilo: Dentro da tag, use o atributo style:
    <h1 style="color: blue; font-size: 24px;"<Meu portifólio CSS</h1>

Tarefa 3: Praticando CSS Interno

O objetivo aqui é criar um arquivo com estilo interno centraliza as regras de uma página no seu cabeçalho.

  1. Crie o arquivo "css_interno.html" dentro da pasta chamada "html/css-interno/".
  2. No Codespace do github, use o atalho html:5 para gerar a estrutura básica.
  3. Dentro da seção head, adicione a tag style
  4. Defina a regra: Crie um seletor para o corpo da página:
    body { color: slategray; font-size: 16px; }
  5. Adicione dois paragrafos no elemento body para verificar se ficaram cinzas e com o tamanho correto.

Tarefa 4: Praticando CSS Externo

O objetivo aqui é criar a estrutura para utilizar css externo, a forma profissional que separa totalmente o conteúdo da aparência.

  1. Crie o arquivo de estilo: Na pasta assets/css/, crie o arquivo style.css.
  2. Crie as regras: Adicione a regra para parágrafos no arquivo com extensão .css:
    p { font-family: Arial, sans-serif; font-size: 14px; }
  3. Crie o arquivo HTML: Na pasta html/css-externo/, crie o arquivo css_externo.html
  4. No Codespace do github, use o atalho html:5 para gerar a estrutura básica.
  5. Vincule os arquivos: No elemento head do HTML, use a tag link com o caminho relativo correto para subir as pastas e encontrar o CSS:
    <link rel="stylesheet" href="../../assets/css/style.css">

Tarefa 5: Criação do Menu Principal

O objetivo aqui é criar a página que será a "porta de entrada" do seu site.

  1. Adicione o título principal: <h1>Meu portifólio CSS</h1> no arquivo index.html.
  2. Crie os links de navegação: Abaixo do título, adicione links para os arquivos das tarefas anteriores:
    • <a href="html/css-inline/css_inline.html">CSS inline</a>
    • <a href="html/css-interno/css_interno.html">CSS interno</a>
    • <a href="html/css-externo/css_externo.html">CSS externo</a>

Tarefa 6: Publicação no GitHub Pages

O objetivo aqui é transformar seu código em um site acessível por qualquer pessoa.

  1. Envie todos os seus arquivos para o repositório no GitHub.
  2. No GitHub, acesse a aba Settings (Configurações) do seu repositório.
  3. No menu lateral, clique em Pages.
  4. Em Build and deployment, selecione a branch main e a pasta /(root), depois clique em Save.
  5. Aguarde alguns minutos e o GitHub fornecerá um link (ex: usuario.github.io/portifolio-info1-2026/) onde seu projeto estará online.
Dica Final: Use sempre o comando Ctrl + Espaço no VS Code se precisar de ajuda com os nomes das pastas ao escrever os caminhos.

Avaliação

Conceito A: Excelente

  • Organização e Nomenclatura: A estrutura de pastas segue rigorosamente o padrão profissional: pasta raiz com o index.html, pasta assets com subpastas css e images, e a pasta html com subpastas modulares. Todos os nomes de arquivos e pastas usam apenas letras minúsculas e hífens, sem espaços ou caracteres especiais.
  • Domínio do CSS: As três formas de inserção (Inline, Interno e Externo) foram aplicadas com sintaxe perfeita e nos arquivos corretos. No CSS externo, o uso de caminhos relativos (como ../../assets/css/style.css) está correto, demonstrando compreensão de hierarquia de diretórios.
  • Funcionalidade: O menu no index.html funciona perfeitamente, levando a todas as subpáginas.
  • Publicação: O projeto está online no GitHub Pages e totalmente funcional.

Conceito B: Bom / Satisfatório

  • Organização: Todas as pastas e arquivos solicitados foram criados e o projeto está completo. CSS: As regras de estilo (azul/24px, slategray/16px, Arial/14px) funcionam conforme o esperado em todos os arquivos.
  • Nomenclatura: Pequenos desvios nas boas práticas (como uma pasta com letra maiúscula), mas que não impedem o funcionamento dos links.
  • Funcionalidade: O menu principal e a publicação estão ativos, embora possa haver pequenos problemas de indentação ou organização visual no código.

Conceito C: Regular / Em Desenvolvimento

  • Pendências: Uma das tarefas (como o CSS externo ou a organização da pasta assets) está incompleta ou contém erros significativos.
  • Erros de Sintaxe: O aluno demonstra confusão entre seletores e propriedades (ex: usar font-color em vez de color) ou esquece de fechar chaves {}.
  • Links Quebrados: Alguns links no index.html não funcionam devido a erros nos caminhos relativos, embora os arquivos existam no repositório.
  • Organização: O projeto carece de modularidade, como colocar o CSS externo na mesma pasta do HTML em vez de na pasta /assets/css.

Conceito D: Insuficiente

  • Incompletude: Faltam duas ou mais tarefas principais, ou o repositório não segue a estrutura mínima exigida.
  • Falha Crítica: Os estilos não carregam em nenhuma página devido a erros graves de vinculação (href incorreto) ou ausência da tag link.
  • Organização Inexistente: Arquivos jogados na raiz sem critério, ignorando a "regra de ouro" de separar conteúdo de estilo.
  • Publicação: O projeto não foi publicado no GitHub Pages ou o repositório está privado, impedindo a avaliação.
Resumo da Avaliação:Para atingir o Conceito A, o aluno deve focar na separação clara entre conteúdo e apresentação e na precisão dos caminhos de arquivos, garantindo que o portfólio seja uma vitrine organizada e funcional do seu aprendizado.
